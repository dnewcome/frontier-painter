# Frontier Painter ŌĆö Slice 01: "Handhold" ŌĆö Official First Demo

This is the official first playthrough demo of **Slice 01**: a free-floating,
zero-g player draws a stroke that freezes into a collidable **handhold**, grabs
it, and pulls themselves hand-over-hand along its centerline to reach the goal.

- Run label: `slice-01-handhold`
- Generated by: `RUN_LABEL=slice-01-handhold npm run playthrough`
- Capture: headless Chromium (Playwright) driving the deterministic `window.game`
  automation API directly (no synthetic pointer/keyboard input), recording a
  webm and one labeled screenshot per beat, then transcoding to `demo.mp4` /
  `demo.gif` with ffmpeg.
- **Result: `goalReached === true`** (and verified to reproduce ŌĆö see B9).

- Animated overview: [`demo.gif`](./demo.gif)

## Scene legend (what you see on screen)

- **Blue ellipsoid** ŌĆö the player (kinematic zero-g controller; collision
  ellipsoid `[0.4, 0.4, 0.4]`). Spawns at `[0, 1, -8]`.
- **Green emissive sphere** ŌĆö the goal at `[0, 1, 8]`; glows green while idle and
  turns **gold** when reached. Win triggers within `goalRadius = 1.0 m`.
- **Amber/orange tube** ŌĆö a frozen handhold created from a drawn stroke
  (`handholdRadius = 0.12 m`).
- **HUD (top-left text)** ŌĆö live `getState()` mirror: ready, position, velocity,
  grabbing/grabT, handhold count, distance-to-goal, elapsed sim time. The green
  **"GOAL REACHED ŌĆö WIN!"** banner appears on win.

The demo camera frames the whole room; the player travels from the spawn corner
(near the tube start) across to the goal corner (near the tube end).

## Reproducing

```bash
RUN_LABEL=slice-01-handhold npm run playthrough
```

The run is idempotent: the output directory is wiped and regenerated each time,
and the playthrough exits non-zero unless the goal is actually reached.
